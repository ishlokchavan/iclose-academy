# iClose Academy — Claude System Context 

## Product Philosophy

This is NOT a generic LMS platform.

This is a premium specialist intelligence and authority platform for real estate micro-market expertise.

The product should feel like:

* Apple-designed specialist software
* premium operator tooling
* structured intelligence infrastructure

NOT:

* a noisy course marketplace
* a creator economy app
* a gamified education platform

Users should feel:
“I’m accessing elite specialist intelligence.”

NOT:
“I’m watching random courses.”

---

# Core Roles

## Learners

## Educators

Real-world operators and specialists:

* community specialists

* luxury specialists

* leasing specialists

## Admins

Internal operators managing approvals and platform operations.

# Architecture Rules

Always:

* use feature-based architecture
* keep files modular
* avoid giant components
* build reusable abstractions
* prioritize server components
* optimize for scalability
* maintain accessibility
* maintain responsiveness

---

# Folder Structure Philosophy

Prefer:

* /features/*
* /components/ui
* /lib/*
* /hooks/*
* /types/*

Avoid:

* chaotic page-level component nesting
* duplicated business logic
* monolithic architecture
